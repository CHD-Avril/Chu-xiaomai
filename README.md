# CHU-小麦点歌台

**CHU-小麦点歌台** 是长大校园广播点歌活动的线上征集入口。

它为同学们提供一个轻量、直接、好用的点歌空间：在开放征集期内，大家可以提交想听的歌曲，为喜欢的投稿点赞，也可以通过当前榜单看到本期校园歌单正在被怎样共同塑造。对广播台来说，它则是一个完整的线上运营台，可以发布公告、设置征集时间、归档历史歌单，并快速整理最终播放名单。

## 我们想做什么

校园广播里的点歌，本质上不只是“收集歌名”。它更像是一段小小的公共情绪：有人把一首歌投给朋友，有人想在放学路上听到熟悉的旋律，也有人只是想让某个瞬间被广播记住。

楚小麦点歌台希望把这件事变得更清楚、更轻松，也更有参与感。

同学不需要加入群聊、不需要私信工作人员，只要打开页面，就能完成投稿和点赞；广播台也不需要在聊天记录、表格和截图之间来回整理，只需要在后台开启一期征集，就能等待大家把歌单填满。

## 网站体验

进入网站后，页面会展示当前点歌征集期的状态。

如果征集正在开放，同学可以填写歌曲名和歌手名完成投稿；提交后的歌曲会出现在歌单列表里，并和其他投稿一起接受点赞。歌单支持搜索、分页和排序，既可以看最热门的歌曲，也可以快速找到自己关心的投稿。

征集结束后，当前歌单会进入公示状态。页面保留结果展示，停止继续投稿和点赞，让本期歌单自然沉淀下来。

## 广播台后台

管理员可以在同一个网站中进入工作台，完成一次点歌活动所需要的主要操作：

- 开启或调整当前征集期
- 发布面向同学的公告
- 归档已经结束的歌单
- 回看历史征集期
- 复制或下载最终歌单

这些能力让点歌活动从“临时收集”变成一个可持续维护的流程，也让每一期歌单都有清晰的开始、结束和记录。

## 项目特点

楚小麦点歌台不是一个复杂的音乐平台，也不是面向商业场景的投票系统。它的目标很明确：服务校园广播台的真实工作流。

因此它尽量保持轻量：

- 页面直达投稿，不做多余跳转
- 点赞和榜单围绕当前征集期展开
- 公告、归档、导出都为广播台日常运营服务
- 适配手机和电脑，方便同学访问，也方便工作人员现场管理

## 技术实现

网站采用静态前端实现，使用 Supabase 保存征集期、歌曲投稿、点赞记录和公告内容。这样的结构部署成本低，维护压力小，适合校园社团或广播台长期使用。

当前项目主要由以下部分组成：

- `public/index.html`：网站页面
- `public/styles.css`：视觉样式
- `public/app.js`：交互与数据逻辑
- `supabase-schema.sql`：数据库结构和权限策略

## 项目定位

楚小麦点歌台属于校园广播台的线上互动工具。

它关心的不是把功能做得多复杂，而是让一次点歌活动顺畅发生：同学愿意投稿，歌曲可以被看见，热门选择能够浮现，广播台可以高效整理，最后让一份带着校园气息的歌单真正被播放出来。
