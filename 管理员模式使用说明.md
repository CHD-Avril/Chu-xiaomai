# 管理员模式使用说明

## 当前登录方式

管理员后台已经改为 Supabase Auth 邮箱登录链接，不再在前端保存管理员账号或密码。

管理员权限由数据库表 `public.admin_users` 控制：只有邮箱存在于 `admin_users.email` 中的用户，完成邮箱验证登录后才可以管理征集期和公告。

## 首次配置

1. 在 Supabase SQL Editor 执行完整的 `supabase-schema.sql`。
2. 在 SQL Editor 执行下面的授权语句，把邮箱替换成实际管理员邮箱：

```sql
insert into public.admin_users (email)
values ('admin@example.com')
on conflict (email) do nothing;
```

被授权的人不需要你提前在 Supabase Auth 里创建用户；他在网站管理员入口输入邮箱后，会收到 Supabase 发送的登录邮件，点击邮件后完成身份验证。

## 使用方式

1. 打开广播台歌单征集页面。
2. 点击“管理员模式”。
3. 输入已写入 `admin_users` 的管理员邮箱。
4. 打开邮箱中的登录邮件完成验证。
5. 登录成功后，可以设置征集期、发布/关闭公告、归档歌单和导出歌单。

## 安全说明

- 前端不提供注册、申请管理员或自助开通入口。
- 不要再向 `public/supabase-config.js` 添加 `adminAccounts` 或任何明文密码。
- 管理员授权只通过 `admin_users.email` 白名单完成。
- 不要再执行旧版 `supabase-rls-setup.sql` 或旧版 `supabase-anti-cheat.sql`；这两个文件已废弃，实际安全策略在 `supabase-schema.sql`。
- 匿名用户只能读取公开数据，投稿、点赞状态读取和点赞切换都通过 RPC 完成。
- 征集期和公告的新增、更新、删除由 RLS 限制为 `admin_users` 中的 Supabase Auth 用户。

## 常见问题

### 登录成功后仍提示没有管理员权限

确认管理员邮箱已经通过上面的 `insert into public.admin_users ...` 语句授权，并且登录时输入的邮箱完全一致。

### 收不到登录邮件

检查 Supabase Authentication 的 Email provider 和邮件模板设置；如果项目关闭了邮件登录或 SMTP 配置异常，登录邮件不会发送。

### 需要新增管理员

直接为新邮箱执行一次授权 SQL。
