# 管理员模式使用说明

## 当前登录方式

管理员后台已经改为 Supabase Auth 邮箱/密码登录，不再在前端保存管理员账号或密码。

管理员权限由数据库表 `public.admin_users` 控制：只有已登录用户的 `auth.uid()` 存在于 `admin_users.user_id` 中，才可以管理征集期和公告。

## 首次配置

1. 在 Supabase SQL Editor 执行完整的 `supabase-schema.sql`。
2. 在 Supabase Dashboard > Authentication > Providers > Email 中关闭公开注册入口，只保留主管理员手动创建用户。
3. 在 Supabase Dashboard > Authentication 中创建管理员邮箱/密码账号。
4. 回到 SQL Editor，执行下面的授权语句，把邮箱替换成实际管理员邮箱：

```sql
insert into public.admin_users (user_id, email)
select id, email
from auth.users
where email = 'admin@example.com'
on conflict (user_id) do update set email = excluded.email;
```

## 使用方式

1. 打开广播台歌单征集页面。
2. 点击“管理员模式”。
3. 输入已在 Supabase Auth 创建并写入 `admin_users` 的管理员邮箱和密码。
4. 登录成功后，可以设置征集期、发布/关闭公告、归档歌单和导出歌单。

## 安全说明

- 前端不提供注册、申请管理员或自助开通入口。
- 不要再向 `public/supabase-config.js` 添加 `adminAccounts` 或任何明文密码。
- 不要开启 Supabase Auth 的公开注册；管理员账号应由主管理员手动创建。
- 不要再执行旧版 `supabase-rls-setup.sql` 或旧版 `supabase-anti-cheat.sql`；这两个文件已废弃，实际安全策略在 `supabase-schema.sql`。
- 匿名用户只能读取公开数据，投稿、点赞状态读取和点赞切换都通过 RPC 完成。
- 征集期和公告的新增、更新、删除由 RLS 限制为 `admin_users` 中的 Supabase Auth 用户。

## 常见问题

### 登录成功后仍提示没有管理员权限

确认管理员邮箱已经通过上面的 `insert into public.admin_users ...` 语句授权，并且邮箱与 Supabase Auth 用户完全一致。

### 忘记管理员密码

在 Supabase Dashboard > Authentication 里重置该用户密码，不需要修改前端代码。

### 需要新增管理员

先在 Supabase Auth 创建新用户，再为该邮箱执行一次授权 SQL。
