# 管理员模式使用说明

## 功能概述

管理员模式允许广播台管理员设置和管理公告，公告会在用户首次访问页面时以弹窗形式显示。

## 使用步骤

### 1. 数据库配置

首先在 Supabase 中创建公告表：

1. 登录到你的 Supabase 项目：https://aiblivjuhccwcwwrwxsl.supabase.co
2. 进入 SQL Editor
3. 复制 `supabase-schema.sql` 文件中关于公告表的部分并执行：

```sql
-- 公告表
create table if not exists public.announcements (
  id uuid primary key default gen_random_uuid(),
  title text not null,
  content text not null,
  is_active boolean not null default true,
  created_by text not null,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

create index if not exists announcements_is_active_idx on public.announcements (is_active);
create index if not exists announcements_created_at_idx on public.announcements (created_at desc);

alter table public.announcements enable row level security;

drop policy if exists "announcements_select_all" on public.announcements;
create policy "announcements_select_all"
on public.announcements
for select
to anon
using (true);

drop policy if exists "announcements_insert_all" on public.announcements;
create policy "announcements_insert_all"
on public.announcements
for insert
to anon
with check (true);

drop policy if exists "announcements_update_all" on public.announcements;
create policy "announcements_update_all"
on public.announcements
for update
to anon
using (true)
with check (true);
```

5. 验证成功后，会显示管理员控制台

### 3. 设置公告

在管理员控制台中：

1. **公告标题**：输入公告的标题，例如"每日播放规则"
2. **公告内容**：输入详细的公告内容，例如：
   ```
   1. 每天最多播放两首外语歌
   2. 禁止播放的歌曲类型：
      - 含有不当内容的歌曲
      - 过于吵杂的摇滚乐
      - 时长超过6分钟的歌曲

   4. 投稿时请填写正确的歌名和作者
   ```

### 4. 禁用公告

如果暂时不需要显示公告：
1. 在管理员控制台中点击"禁用公告"按钮
2. 确认后，用户将不再看到公告弹窗

## 公告显示逻辑

- 用户首次访问页面时，如果有活跃公告，会弹出公告弹窗
- 用户点击"我知道了"或关闭按钮后，公告不会再次显示（使用 localStorage 记录）
- 如需重新查看公告，用户可以清除浏览器数据或使用隐私模式

## 注意事项

1. **安全性**：当前的密码验证是前端简单验证，适合内部使用。如果需要更高的安全性，建议实现后端认证系统。
2. **公告更新**：每次保存新公告时，旧公告会自动禁用
3. **内容格式**：公告内容支持换行，会在弹窗中保留格式
4. **字符限制**：标题最多100字符，内容最多1000字符

## 故障排除

### 问题：公告保存失败
- 检查 Supabase 连接是否正常
- 确认数据库表已正确创建
- 查看浏览器控制台的错误信息

### 问题：公告不显示
- 确认公告已启用（is_active = true）
- 清除浏览器 localStorage 后刷新页面
- 检查是否有多个公告，系统只显示最新的一个

### 问题：账号或密码错误
- 确认输入的账号和密码正确（注意大小写）
- 检查 `public/supabase-config.js` 中的管理员账户配置
- 默认账号：`changdaxiaomaijun`，密码：`gbt666`

## 技术支持

如有问题，请检查：
1. 浏览器控制台是否有错误
2. Supabase 数据库连接是否正常
3. 公告表是否正确创建
