1. **Windows/Linux**: 按 `F12` 或 `Ctrl + Shift + I`
2. **Mac**: 按 `Cmd + Option + I`
3. 或者：右键页面 → "检查" → 选择 "Console" 标签

按 `Ctrl + F5` (Windows) 或 `Cmd + Shift + R` (Mac) 强制刷新

## ❌ 常见错误及解决方案

### 错误 1: 配置无效

**控制台显示：**
```
❌ 错误: Supabase 配置无效
请检查 public/supabase-config.js 文件
```

**状态显示：** "配置无效"（红色）

**解决方案：**
1. 打开 `public/supabase-config.js`
2. 检查 URL 和 Anon Key 是否正确
3. 确保格式正确（参考下面的配置模板）

---

### 错误 2: 查询歌曲失败

**控制台显示：**
```
❌ 查询歌曲失败: {message: "relation 'public.songs' does not exist", ...}
```

**原因：** 数据库表不存在

**解决方案：**
1. 登录 Supabase: https://app.supabase.com
2. 进入 SQL Editor
3. 执行 `supabase-schema.sql` 中的全部 SQL
4. 刷新页面

---

### 错误 3: 查询公告失败

**控制台显示：**
```
❌ 查询公告失败: {message: "relation 'public.announcements' does not exist", ...}
公告表可能不存在或未创建
```

**原因：** 公告表未创建（不影响主要功能）

**解决方案：**
- 如果不需要公告功能，可以忽略
- 如果需要，执行以下 SQL：

```sql
create table if not exists public.announcements (
  id uuid primary key default gen_random_uuid(),
  title text not null,
  content text not null,
  is_active boolean not null default true,
  created_by text not null,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

create index if not exists announcements_is_active_idx on public.announcements (is_active);
create index if not exists announcements_created_at_idx on public.announcements (created_at desc);

alter table public.announcements enable row level security;

create policy "announcements_select_all" on public.announcements for select to anon using (true);
create policy "announcements_insert_all" on public.announcements for insert to anon with check (true);
create policy "announcements_update_all" on public.announcements for update to anon using (true) with check (true);
```

### 错误 4: 网络连接失败

**控制台显示：**
```
❌ 数据库连接失败！
错误类型: FetchError / TypeError
错误消息: Failed to fetch / NetworkError
```

**原因：** 
- 网络问题
- Supabase 服务停机（免费版每周停机 1 小时）
- 防火墙阻止

**解决方案：**
1. 检查网络连接
2. 访问 https://aiblivjuhccwcwwrwxsl.supabase.co 确认服务正常
3. 等待几分钟后重试
4. 检查防火墙设置

---

### 错误 5: 权限错误

**控制台显示：**
```
❌ 查询歌曲失败: {message: "new row violates row-level security policy for table 'songs'", ...}
```

**原因：** RLS 策略配置错误

**解决方案：**
1. 登录 Supabase
2. 进入 Authentication → Policies
3. 确保 songs、song_likes、announcements 表都有正确的策略
4. 或重新执行 `supabase-schema.sql`

## 📋 配置检查清单

### 1. 检查配置文件

打开 `public/supabase-config.js`，确认：

```javascript
export const supabaseConfig = {
  url: "https://aiblivjuhccwcwwrwxsl.supabase.co",  // ✅ 必须有
  anonKey: "eyJhbGci...",  // ✅ 必须是很长的 JWT token
  tables: {
    songs: "songs",
    likes: "song_likes",
    announcements: "announcements",
  },
};
```

### 2. 检查数据库表

在 Supabase 中确认以下表存在：
- [ ] `songs` 表
- [ ] `song_likes` 表
- [ ] `announcements` 表

### 3. 检查网络连接

- [ ] 能正常访问互联网
- [ ] 能访问 Supabase 控制台
- [ ] 没有被防火墙阻止
